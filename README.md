# Waybill

Проект состоит из двух Spring Boot модулей: `producer` и `consumer`.

Важные особенности:

- используется составной индекс/уникальный ключ по паре `invoice_number` + `invoice_date`;
- используется SOAP для получения полного документа из `producer`;
- есть преобразование XML через XSLT-преобразование. Выбор XSLT-шаблона доступен пользователю на экране интерактивно;
- XSLT-шаблоны хранятся в БД, пользователь может добавить новые и обновить имеющиеся.

## Модули

### producer

`producer` отвечает за хранение и публикацию документов УПД.

Основные функции:

- импорт документа из Excel через REST endpoint;
- хранение документа, строк документа, продавца, покупателя и товарных позиций в PostgreSQL;
- поиск документа по составному ключу `invoice_number` + `invoice_date`;
- публикация списка заголовков документов для `consumer`;
- публикация SOAP endpoint, через который `consumer` получает полный документ.

Основные endpoints:

- `POST /api/import/waybill` - импорт Excel-файла;
- `GET /api/waybill/documents/headers` - список заголовков документов;
- `POST /soap/waybill` - SOAP endpoint для получения документа;
- `GET /soap/waybill?wsdl` - WSDL SOAP-сервиса.

По умолчанию модуль работает на порту `8080` и использует схему БД `waybill_producer`.

### consumer

`consumer` отвечает за пользовательский просмотр и анализ документов, полученных из `producer`.

Основные функции:

- загрузка списка документов из `producer`;
- переход к анализу выбранного документа;
- запрос полного документа у `producer` по `invoiceNumber` + `invoiceDate`;
- получение XML через SOAP;
- преобразование XML в HTML через выбранный XSLT-шаблон;
- хранение XSLT-шаблонов в PostgreSQL;
- загрузка новых XSLT-шаблонов и обновление существующих.

Основные endpoints:

- `GET /list/waybill-documents` - список документов;
- `GET /import/waybill-document` - экран анализа выбранного документа;
- `GET /api/xml/waybill-document` - XML выбранного документа;
- `GET /api/xsd/waybill-document` - JSON-представление выбранного документа;
- `GET /xslt` - список XSLT-шаблонов;
- `GET /xslt/{fileName}` - содержимое XSLT-шаблона;
- `POST /xslt` - загрузка или обновление XSLT-шаблона.

По умолчанию модуль работает на порту `8082` и использует схему БД `waybill_consumer`.

## Основной сценарий

1. Пользователь импортирует документ в `producer`.
2. `producer` сохраняет документ и обеспечивает уникальность по `invoice_number` + `invoice_date`.
3. Пользователь открывает в `consumer` список документов: `http://localhost:8082/list/waybill-documents`.
4. В списке пользователь нажимает `Анализ`.
5. `consumer` запрашивает полный документ у `producer` через SOAP по номеру и дате счета.
6. Полученный XML преобразуется в HTML выбранным XSLT-шаблоном.
7. Пользователь может интерактивно переключать XSLT-шаблон на экране анализа.

## Запуск

Требования:

- Java 25;
- Maven;
- PostgreSQL.

Сборка:

```bash
mvn test
```

Запуск `producer`:

```bash
mvn -pl producer spring-boot:run
```

Запуск `consumer`:

```bash
mvn -pl consumer spring-boot:run
```

Конфигурация подключения к БД задается через `application.properties` и может быть переопределена переменными окружения.
